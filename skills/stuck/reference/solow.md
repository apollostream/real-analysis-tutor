# Level 1 — Solow's Forward-Backward Method

Source: CG Part III A. This is the primary procedural framework for every proof attempt, and it is always Level 1 — run it first, before any Level 2 escalation, on every stuck episode without exception. It applies not only to constructing proofs but to understanding definitions and theorems before any proof work begins.

Run these passes with the learner conversationally, one question at a time. Do not answer your own questions — the learner works each pass.

## Applying the method to definitions

**Backward pass — unpack the structure.** Ask what the definition is actually asserting. Identify the key terms; ask the learner to rephrase the definition in their own words; ask what logical form it takes (conjunction? implication? quantified statement?).

**Forward pass — connect to known ground.** Ask what examples the learner already knows should satisfy this. Ask what should NOT satisfy this. Ask what simpler or related definition this generalises or specialises.

## Applying the method to theorems (before proving them)

**Backward pass — understand the conclusion.** Ask the learner to identify the conclusion's form (existence claim? equality? implication? bound?). Ask what technique or object would be needed to make this true. Ask what quantifier structure the conclusion has.

**Forward pass — interrogate the hypotheses.** Have the learner list every hypothesis explicitly — stated and unstated. For each hypothesis, ask:
- Why is it there?
- What breaks without it?
- What does it immediately let you conclude?

## Section-level application: build the map before the territory

Before the learner attempts a single proof in a section, run the Forward-Backward method over the entire section first, not just the current theorem:

1. Start from the END — find the section's main result and ask what it is claiming.
2. Work backward through the section from there.
3. Then start from the BEGINNING with the destination in mind.

This turns mathematical reading from a linear march into building a map before exploring the territory. By the time the learner sits down to prove exercises, most of the hard conceptual work is already done: they know what the section is fundamentally about, which definitions are critical, how the theorems relate, and where the logical pressure points are.

## When to escalate

If the learner has genuinely run both passes — backward and forward — on the definition or theorem at hand and is still unable to move, Level 1 has stalled. Escalate to Level 2 (`reference/dlmk-polya.md`) and increment the stuck counter for the concept as described in `SKILL.md`. Do not escalate merely because the learner hasn't tried; if a pass hasn't actually been run, run it before moving on.
